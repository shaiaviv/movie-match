Runtime Flow — From Zero to Match

1
Browser loads → React mounts
main.jsx creates the React root, wraps in BrowserRouter. App.jsx routes / → Home, /room/:roomId → Room.
2
Socket singleton created (not connected)
socket.js creates the Socket.io client with autoConnect: false. Actual connection waits until Home's useEffect calls socket.connect().
3
Creator selects filters → emits create-room
Home.jsx builds a filter payload (genre, year range, runtime, rating, language, certification) and emits create-room to the server.
4
Server fetches 20 movies from TMDB
index.js calls tmdb.js → builds Discover API URL → fetches 3 random pages in parallel via Promise.all → shuffles → returns top 20.
5
Room created in memory
roomManager.js generates a 6-letter code (from unambiguous charset), stores the room in a Map, optionally persists to db.js Postgres.
6
Client navigates to /room/:roomId
Home.jsx receives room-created, saves movies to sessionStorage, and navigates. Room.jsx mounts and immediately emits join-room to register its socket.
7
Second player joins
Joiner enters the code or navigates to the URL. index.js calls joinRoom() → adds joiner to room → emits partner-joined to both sockets.
8
Both players swipe → votes recorded server-side
MovieCard.jsx detects drag/flick gestures → calls onVote(liked) → Room.jsx emits vote → roomManager.js records it and checks for mutual like.
9
Both players finish → matches revealed
When each player swipes the last card, they emit player-done. When both are done, roomManager.js returns the accumulated matches array, and all-done is broadcast to the room.

How to Run Locally

Terminal — setup & run
# Prerequisites: Node.js 18+, free TMDB API key from themoviedb.org
 
$git clone https://github.com/shaiaviv/movie-match.git
$cd movie-match
 
# Create the environment file
$echo "TMDB_API_KEY=your_key_here" > server/.env
 
# Install all dependencies (root + server + client)
$npm install
$npm install --prefix server
$npm install --prefix client
 
# Start both servers with one command
$npm run dev
 
[server] MovieMatch server running on http://localhost:3001
[client] VITE v5.x.x ready in 234 ms
[client] ➜ Local: http://localhost:5173/
 
# Open two tabs to test the two-player experience
# Tab 1: Create a room. Tab 2: Enter the code to join.
 
# Common errors:
# ⚠ TMDB_API_KEY not set → create server/.env with your key
# Error: Cannot find module 'socket.io' → run npm install --prefix server

Quick Function Reference

Function File What it does Returns
generateCode() roomManager.js Picks 6 random chars from unambiguous charset. Recursive if code exists. string (6 chars)
createRoom(socketId, movies) roomManager.js Generates code, initialises room object in Map, optionally persists to DB. Promise<string> (roomId)
joinRoom(code, socketId) roomManager.js Finds room (memory then DB). Validates capacity. Adds joiner. Promise<{room} | {error}>
getRoom(code) roomManager.js Simple Map lookup by room code. room | null
getRoomBySocket(socketId) roomManager.js Linear scan of all rooms to find one containing a given socket ID. room | null
recordVote(roomId, socketId, movieId, liked) roomManager.js Stores vote. If liked AND other player already liked → pushes to matches. void
markDone(roomId, socketId) roomManager.js Adds player to done Set. Returns matches only when ALL players are done. matches[] | null
removeUser(socketId) roomManager.js Removes socket from room. Schedules 2hr TTL cleanup if room empty. room | null
shuffle(arr) tmdb.js Fisher-Yates in-place shuffle. Ensures random movie order per room. arr (mutated)
randomPages(count, max) tmdb.js Picks count unique random page numbers 1..max using a Set. number[]
formatMovies(results) tmdb.js Filters out incomplete entries, maps TMDB fields to app shape. movie[]
fetchMovies(apiKey, genreId, opts) tmdb.js Builds TMDB Discover URL, fetches 3 pages in parallel, shuffles, returns 20. Promise<movie[]>
initSchema() db.js Creates rooms table if it doesn't exist. No-op if no DATABASE_URL. Promise<void>
handleCreate() Home.jsx Reads filter state, emits create-room with undefined-cleaned payload. void
handleJoin(e) Home.jsx Normalises input (accepts full URL or code), emits join-room. void
handleVote(liked) Room.jsx Emits vote, advances card index, emits player-done on last card. void
triggerExit(liked) MovieCard.jsx Guards against double-trigger, sets exit animation, calls onVote after 350ms. void
finish(endX, elapsed) MovieCard.jsx Evaluates if drag was long enough or fast enough to count as a swipe. void
